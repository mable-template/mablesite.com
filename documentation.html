1.0

General Document Setup

A typical Mable HTML document will look like:

<html>
	<head>
		<!-- Page Info -->
		<title>This is a sample Mable HTML Header</title>
		<meta name="description" content="Page Description">
		<meta name="keywords" content="Page, Tags">
		<meta property="og:image" content="https://full-url">
		<link href="images/brand/favicon.png" rel="shortcut icon">
		<meta name="viewport" content="width=device-width, initial-scale=1.0">

		<!-- CSS Sheets -->
		<link type="text/css" rel="stylesheet" href="css/style.css">
			
		<!-- Themes -->
		<link type="text/css" rel="stylesheet" href="themes/sample-theme" id="custom-theme-sheet">

		<!-- Packages -->
		<link type="text/css" rel="stylesheet" href="packages/sample-package.css">
	</head>
	<body>
		// Sections
	</body>
</html>

Simply put all of that into a .html document, change the href fields if the document is in any subfolder(s), and change the data to your liking. A lot of the stuff in the <head> section isn't neccicary, and is just good practice to have. Remove anything you don't need.

2.0

Layout

This section is about the large blocks that make up Mable webpages.

2.1

Dynamic Screen Sizes

In order to help compensate for the wide range of screen sizes, Mable has implemented compatibility for Bootstrap-standard dynamic screen size selectors:

  • xs: width ≤ 576px
  • sm: width ≤ 768px
  • md: width ≤ 992px
  • lg: width ≥ 992px
  • xl: width ≥ 1200px

Whenever you see any one of those five in a class name, the other four will work.

2.2

Sections

Generally, the page is comprised of sections, one after the other, which looks like a series of horizontal stripes. They are typically coded like this:

<!-- Section Name -->
<section>
	<div class="container">
		<div class="row">
			// Regular HTML content goes here
		</div>
	</div>
</section>

Containers are used to dictate the width of the section's content. The three classes to use are container for regular text, container-wide for a wider area, and container-flud for an area with no maximum width.

A row is not strictly speaking neccicary, but is useful for separating different sections of content. A small bit of margin is added between rows. It is also very useful for grids, and it makes adding new rows easier after the section is initially made.

2.3

Grids

Grids are very simple, and are similar to Bootstrap's. To create a grid, create a parent element with the grid class. It is typical to use a row for the parent element, but you can techically use anything.

<section>
	<div class="container">
		<div class="row grid">
			// Child elements
		</div>
	</div>
</section>

Child elements are then allowed to have a specified width (in columns) from 1 to 12.

<div class="row grid">
	<div class="col-6">
	</div>
	<div class="col-3">
	</div>
	<div class="col-2">
	</div>
</div>

These child elements can also have their width dynamically resized based off the screen width.
